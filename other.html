Custom TF2 Portraits

8/2022

Tools used: SFM, paint.net, VTFEdit

In-game screenshot of the new Red Pyro portrait
SFM render of the Blue Spy's portrait
Side-by-side comparison of some of the old portraits and the new portraits
A look into the .VPK archive for the custom files using GCFScape (73 new files)

When I played more TF2 I always used the option to replace the default class portraits with a live-3D rendered version. It then occurred to me that rendering a live 3D model over using the default 2D sprites costs performance.

However, one cool feature of the 3D portraits is that they retain customizations applied to your character, while the 2D versions didn't. So I set out to rectify this issue, not to mention that the defaults already had problems.

Since SFM cannot render with transparency by default, I had to devise a new method to get accurate transparency on the renders. I achieved this by having two sets of lights, one for the normal render and another for creating a "fake alpha" channel, which floods the character models with light over a completely black background, any irregularities in the alpha render could be manually rectified by hand and contrast settings.

A collection of various works I have completed. Was initially going to be hosted using carrd, but there were a couple of complications, including a 50 element limit.

I initially learned HTML and CSS in 2021-2022 during one of my modules, but I hadn't utilized it until this project and not to this scale. So, I had to relearn it along with JavaScript (which I dabbled in with the incremental game bitburner) to add interactive elements.

One element I'm particularly proud of are the image galleries which use mainly use arrays to deactivate and reactivate the appropriate page elements.

Another aspect is that this website is constructed using all my code (or code I learned from tutorials), rather than using a template to construct from. To ensure it is all truly "my own" work.
