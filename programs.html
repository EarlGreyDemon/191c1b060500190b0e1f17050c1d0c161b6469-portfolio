DVD Corner Thingy

11/2022

Languages/Engines used: Processing & Java

Screenshot taken after leaving the program for a while
Screenshot taken shortly after start
I left the program running for far too long
A screenshot of some of the code

Once, I was bored during a lecture, so I booted up Processing and made an application resembling DVD screensavers of old.

Uses Vectors and a changing hue to produce the effect, additionally no background is drawn, so any trail remains after being drawn. Vectors bounce when hitting the edge of a screen, additionally a random directional vector is generated on startup, so each image has variance.

Receipt App

1/2024

Languages/Engines used: C# & WPF

Tools used: Visual Studio

An .XAML for a custom user control I made
The .XAML for the application's main UI
The application's UI (still work-in-progress)

I initially came up with the idea of this application in 2023, since I felt I was being robbed by the local Tescos. Since then, the draft has been revised from a console application to a Winforms app to a WPF app.

The purpose of this application is to store data on grocery shopping, so that information can be compared and users can get the best prices from their shopping trips.

Initially I was going to make it using a console line interface, but I figured that it'd be difficult to use. During one of my modules we learned Winforms so I was going to use it, but after being driven insane by repeatedly developing applications using it for the same module, I settled on WPF.

As a bonus I get to learn a new toolkit. WPF seems far more flexible and powerful, so going forward if I make more applications, it will likely be my go-to.

Starfall

9/2021

Languages/Engines used: Python & Pygame

Tools used: IDLE

Screenshot #1
Screenshot #2
Screenshot #3

On one of my old Samsung smartphones there was a live wallpaper, (I think it was called "Network") which had coloured cubes fly onto screen leaving behind trails of the same colour, so I tried to create something similar with Python.

The application currently spawns randomly-coloured main cubes, which spawn their own trail objects as they fall down.

In an attempt to make it run better, I also ported it to another Python library: Pyglet, but unfortunately the new version ran very poorly.

Custom Vector / Matrix classes

5/2024

Languages/Engines used: C++ and OpenGL

Tools used: Visual Studio

A screenshot of an OpenGL application using my vectors and matrices
A snippet of my rotation matrix calculator, converting eulerian angles into quaternions
Some methods I created as a part of my three-element vector class
A snippet of the .obj importer I created, works using my vector and matrix classes

One of my modules of work was Graphics, which had us working in C++ and with OpenGL. The assessment for said module was to develop a graphics program, and the highest marking criteria included creating and using a custom-made library for vector and matrix calculations instead of OpenGL's own.

In addition, I also wrote a basic .obj reader, which works in conjunction with my vectors and matrices to allow custom .obj files to be used in an OpenGL application.

Going forward, I would like to make more appropriate use of references and pointers, along with destructors where necessary.
