Untitled Tower Defence Game

11/2022

Languages/Engines used: Processing & Java

Tools used: Blender

Game before first enemy wave
Incoming basic enemies
Regenerating enemies being beaten back
Large wave of incoming basic enemies

During my first year at university, one of my first modules was programming, part of the assessment for this unit was designing and making a "Clicker Game" using Processing.

There were many marking criteria I strove to implement in my project. One feature is the use of .txt files, which stores not only highscore information but also map and enemy wave information too. Because of this, someone could create their own map with custom waves.

There are 5 tower types, and 4 types of enemies included. Each tower and enemy type have slight variation, for example green enemies can recover from damage if not killed quickly, and some tower types fire explosion-like projectiles rather than normal bullets.

Another feature I was able to implement was the ability to use external images for animated sprites, using Blender's freestyle feature, I was able quickly make sprites with simple details and small movements.

Spells of Deceit

1/2024

Languages/Engines used: Unity & C#

Tools used: Visual Studio

Game main menu
The battle screen
The archer has been killed while the knight suffers a debuff

During my second at univeristy year, there was an optional 3-day game jam held just before lessons started back up after the winter break. I formed a group with 3 others, and the theme given was traitors.

The game we made was like a turn-based RPG with 3 members trying to defeat an enemy, but the player character had a secret task of killing their team.

I contributed my programming skills and made the turn system. Each action taken by the player or NPCs sends a struct to the game manager which then executes the actions on the appropriate targets. This sort of system also allowed us to apply debuffs to the NPCs along with the regular damage effects.

Unity Simple Clicker Game

2/2024

Languages/Engines used: Unity & C#

Tools used: Visual Studio

Work in progress screenshot of the main "Production floor"
Various text notifications & current planned screens
Purchases floor, 20 items already purchased

A self-motivated project to learn more about Unity and to see if I could bring some fresh interesting mechanics to incremental games.

Still under development, but currently has passive income items, an upgradeable button and research points which are rewarded randomly for pressing the button. Currently, the costs for purchasing items are scaled up using animation curves and pressing the button instances text particles to give visual feedback.

In the future I want to implement an upgrade tree to spend earned research points on along with more resource management systems.
